# Instagram / Facebook 콘텐츠 검수 체크리스트

이 문서는 카드뉴스 JSON이 완성된 뒤 발행 전에 점검하는 검수 기준이다.
다른 가이드 조각(00~07번)이 "어떻게 쓸지"를 정한다면, 이 문서는 "다 쓴 걸 어떻게 통과/탈락시킬지"를 정한다.

---

## 1. 자동 검수 (기계적으로 확인 가능)

| 항목 | 기준 | 결과 |
|---|---|---|
| 카드 수 | story_plan 3개 → 5장, 4개 → 6장과 실제 cards 배열 길이 일치 | PASS/FAIL |
| 카드 순서 | 1번 cover, 마지막 cta, 나머지 middle | PASS/FAIL |
| 첫 장 필드 | items 빈 배열, highlight_text 빈 문자열, cloud_label 값 존재 | PASS/FAIL |
| CTA 카드 필드 | items 빈 배열, highlight_text 빈 문자열 | PASS/FAIL |
| 중간 카드 title | 줄바꿈 정확히 1번, 각 줄 15자 이내 | PASS/FAIL |
| 중간 카드 subtitle | 줄바꿈 없이 1줄, 25자 이내 | PASS/FAIL |
| 첫 장/CTA 카드 title·subtitle | title 각 줄 15자 이내, subtitle 줄바꿈 없이 25자 이내 | PASS/FAIL |
| highlight_text | middle 카드에서 title 안에 실제 포함된 문구인지 | PASS/FAIL (참고용 — Figma에서 수동으로 강조 위치를 조정할 수 있어 전체 판정을 막지 않음) |
| items 개수 | layout_type별 규칙 준수 (steps_vertical 2~4, compare_2col 정확히 2, flow_process 3~4, keyword_boxes 3~4, stacked_boxes 2~3) | PASS/FAIL |
| 숫자 문구 일치 | title/subtitle에 "N단계", "N가지"처럼 숫자를 명시했으면 실제 items 개수가 N개와 일치하는지 (예: "개선으로 연결하는 4단계"라고 써놓고 항목이 3개면 FAIL) | PASS/FAIL |
| 후킹 예시문장 재사용 | 1번 카드 title/subtitle이 05-hook-patterns.md의 데모 문장(예: "문의가 반복되면 보통 상담사 탓을 합니다")을 토씨만 바꿔 그대로 쓰지 않았는지 | PASS/FAIL |
| 수치 출처 표기 | 중간 카드에 "N%"·"N배" 같은 구체적 수치가 있으면 그 카드의 cta 필드에 출처가 채워져 있는지 (수치 없는 카드는 검사 제외) | PASS/FAIL |
| 해시태그 개수·필수 태그 | 12~15개이며 #CS쉐어링 #CS대행 #고객센터대행 포함 (06-content-rules.md 기준). 개수 초과 시 자동으로 15개까지 보정, 부족 시 필수 태그부터 채움 | PASS/FAIL |
| 검수용 정보 비노출 | planning, design_point 등 내부 메모가 caption/카드 텍스트에 새어나오지 않았는지 | PASS/FAIL |

이 표는 코드가 JSON을 파싱해서 직접 계산한다. 다만 highlight_text와 중간 카드 title 항목은 실패해도 최종 판정(FAIL)에는 반영하지 않는다 — 둘 다 Figma 작업 단계에서 바로 고칠 수 있는 사소한 항목이기 때문이다.

---

## 2. 정성 평가 (5점 척도)

점수만 매기지 않는다. 각 항목마다 왜 그 점수인지 근거를 대고, 5점 미만이면 구체적으로 어떻게 고치면 되는지 대안까지 함께 제시한다 — 채점자 재량으로 점수만 던지는 것을 방지하기 위함이다.

| 항목 | 체크 기준 | 점수 |
|---|---|---|
| 문장 자연스러움 | 명사구 나열, "단순히 A가 아니라 B" 류 도치·대구 남발, "혁신적인/완벽한/다양한" 같은 뭉뚱그린 형용사, 억지로 맞춘 3단 병렬 구조 — 이 4가지 유형이 0~1개면 5점, 2개 이상이면 감점 | /5 |
| 서비스 범위 정확성 | ① 구체적 수치·사례를 쓸 때 출처가 있거나("확인된 수치는" 등) 가상 예시임이 표시되어 있는지. ② 서비스명·기능(예: "OASIS", "AI StandBy", 쉐어링/전담/토탈/시간제 서비스 등)을 언급했다면 company-facts.md에 실제로 있는 서비스가 맞는지. ③ 수치 자체가 company-facts.md에 있더라도, 그 수치가 뒷받침하는 주장까지 원문과 같은지 반드시 대조한다 — 숫자만 베끼고 인과관계·맥락을 새로 지어내면 숫자가 진짜라도 거짓 주장이다. company-facts.md에 없는 서비스명·기능을 지어냈거나, 수치는 있어도 그 수치가 뒷받침하는 주장이 원문과 다르면 이 항목은 0점 처리 | /5 |
| 고객 문제 반영 | 카드 전체에 걸쳐 "어떤 상황의 담당자가 → 무엇 때문에 → 어떤 영향을 받는지"가 구체적으로 드러나는지 | /5 |
| 채널 적합성 | B2B 담당자(팀장급) 대상 톤 유지, 소비자·개인 셀러 톤으로 새지 않았는지 | /5 |
| 후킹력 | "~하는 이유"/"~인 이유" 같은 문장 틀 자체는 감점 대상이 아니다. 실제로 볼 것은 ① 구체성 — 숫자·장면·대사로 못박았는지, 뭉뚱그린 말("이유가 있습니다")로 퉁치지 않았는지 ② 정보 격차 — 결론은 안 주더라도 그 결(반전 지점·방향)을 미리 살짝 흘렸는지, 아니면 "궁금하죠? 다음 장에서"처럼 완전히 백지상태로 미루는지. 이 두 가지가 다 없으면(구체성도 없고 반전의 결도 안 보이면) 3점 이하로 처리 | /5 |
| CTA 연결성 | 마지막 장이 새 문제를 추가하지 않고, 앞선 카드 내용과 자연스럽게 이어지는 행동 유도로 끝나는지 | /5 |

**기준선**: 정성 평가 합계 24/30 미만이거나, 문장 자연스러움·후킹력 중 하나라도 3점 이하면 재작성.

## 3. 실패 사례 (금지 패턴 예시)

- "성수기 피로감에 더해 비수기 문의의 특성" — 명사구 나열, 의미 연결 끊김 → FAIL
- "다양한 솔루션을 통해 효과적으로 대응할 수 있습니다" — 구체성 없는 형용사 나열 → FAIL
- "문의 유형별 확인 정보가 다릅니다" — 01-direction-story.md에서 이미 금지한 결론형 문장이 카드 본문에 재등장 → FAIL
- 후킹 카드 title이 "OO에 대해 알고 계셨나요?"처럼 04-reference-examples.md의 실제 시리즈 문구와 구조까지 똑같이 재탕된 경우 → FAIL (표현만 바꾼 반복)
- "반복 문의의 진짜 이유"— "이유"라는 단어가 문제가 아니라, 이유의 결(구체적으로 뭐가 반전인지)을 전혀 안 흘리고 "이유가 있다"고만 말해서 구체성도 정보 격차도 없는 경우 → 감점. 좋은 예 "고객이 화내는 건 응대가 늦어서가 아니라, 대답이 매번 달라서입니다"는 똑같이 "이유"를 말하지만 반전 지점을 이미 절반 보여줘서 궁금증을 만든다.
- "김 대리가 이 분석 결과를 공유하자 문의가 40% 감소했습니다"처럼 출처도 없고 가상 표시도 없이 실화·확정 수치처럼 서술 → FAIL
  - 참고: "가령 김 대리가 있는 CS팀이라면, 이런 분석으로 문의를 줄일 수 있습니다"처럼 가상 예시임을 밝히면 PASS. 인물 예시 자체는 문제가 아니다.
- "분석이 멈추지 않으려면 구조에 담아야 합니다"처럼 무엇을(what), 왜(why) 구조에 담아야 하는지 구체적 대상이 빠진 채 "구조/체계/프로세스에 담다" 같은 관념어로만 문장을 마무리 → FAIL. 카드 items가 이미 구체적 단계를 담고 있다면, subtitle은 그 단계를 안 지키면 생기는 구체적 결과(예: "한 단계라도 빠지면 문의가 다시 반복됩니다")로 써야 한다.
- "반복 문의의 70%는 잘못된 운영 프로세스 때문입니다" — company-facts.md에는 "단순 반복 문의 비중이 70~80%"라는 수치만 있을 뿐, "그 반복 문의의 70%가 운영 프로세스 탓"이라는 인과관계는 어디에도 없음 → FAIL. 숫자가 원문에 있다고 안심하지 말고, 그 숫자로 어떤 주장을 하는지까지 원문과 대조한다.

## 4. 최종 판정

| 판정 | 조건 |
|---|---|
| PASS | 1번 자동 검수(비차단 항목 제외) 전부 통과 + 2번 합계 26점 이상 |
| 조건부 PASS | 자동 검수 통과, 정성 평가 22~25점 (경미한 수정 후 발행 가능) |
| FAIL | 자동 검수(비차단 항목 제외) 하나라도 실패, 또는 정성 평가 22점 미만, 또는 3번 실패 사례에 해당 |

비차단 항목(참고용, 전체 판정을 막지 않음): highlight_text_일치, 중간카드_title_줄바꿈 — 둘 다 Figma에서 손으로 바로 고칠 수 있는 사소한 항목이라 재생성까지 가지 않는다.
