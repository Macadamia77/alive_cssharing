# 03. 품질 검증 하네스 v1.3

> **역할**: 원고 저장 전 **반드시 실행**하는 검증 절차. B등급(80점) 미만이면 재작성 대상.
> 수치 기준의 원 출처는 `01-writing-guide.md` 2절이다 — 숫자가 어긋나면 01을 따르고 이 문서를 갱신한다.
> 조립(HTML 변환) 단계는 결정론적 코드이며 품질을 재검증하지 않는다 — **이 검수가 유일한 게이트다.**
> 애매하면 통과시키지 말고 FAIL로 처리한다.

---

## 실행 순서

```
[Step 0] 완성도 게이트 — 1개라도 FAIL이면 즉시 재작업 (점수 무관)
[Step 1] 정량 항목 자동 측정
[Step 2] 절대 금지 스캔
[Step 3] 리캐치 스타일 구조 검증
[Step 4] 정성 항목 셀프리포트
[Step 5] 종합 점수 산출
```

---

## Step 0. 완성도 게이트 (PASS/FAIL)

| 항목 | PASS 조건 |
|---|---|
| 메타 표기 누출 | PUBLISH 블록에 편집지시문·검증결과 등 NOTES 전용 내용 0건 |
| 이미지 마커 | `[IMAGE: ...]` 마커가 본문에 자연스럽게 배치되어 있음 |
| 이미지 수량 | 6~8장 (소제목마다 1장 강제 아님 — 표로 대체 가능한 정보는 이미지 없이 표만 존재해도 됨) |
| 표(Table) 사용 | 비교표·수치 나열형 정보가 이미지 카드가 아니라 본문 마크다운 표로 들어가 있음 |
| 강조 지정 | NOTES 블록 강조 지정 표에 `{{hl:}}` 2개 이상 기록 |
| CTA 타겟 존재 | CTA 2곳 모두 실제 URL/전화번호 또는 `[RICH:...]` 포함 |
| 소제목별 자사 서비스 연결 | 본문 소제목 4~6개 전부 자사 서비스명/기능 언급 1회 이상, 명칭은 `05-brand-cta-reference.md` 카탈로그와 정확히 일치 |
| GEO/AEO 블록 | FAQ/핵심요약/체크리스트 중 1개 이상 존재 + NOTES에 위치 기록 |
| 도입부 선요약 (AEO) | 도입 훅 마지막에 핵심 결론·수치 2~3문장 요약이 있음 (본문을 다 읽어야 답이 나오는 구조 아님) |
| 수치 출처 | 본문에 인용된 수치마다 출처(기관명·시점) 표기 |

**표·GEO/AEO 블록·수치 출처 3개 항목은 원고의 자기보고(NOTES)를 그대로 믿지 말고, PUBLISH 블록
텍스트를 직접 훑어 기계적으로 재확인한다** (표 항목 `|`로 시작하는 행 존재 여부, "FAQ"·"체크리스트"·
"핵심" 등 GEO/AEO 키워드 존재 여부, 수치 뒤에 "~에 따르면"·"~기준" 등 출처 표현이 붙어 있는지). 셋
중 하나라도 실제 텍스트에서 확인 안 되면 NOTES의 자기보고 PASS와 무관하게 FAIL로 처리한다.

## Step 1. 정량 항목

### 글자수·제목

| 항목 | PASS |
|---|---|
| 본문 공백 제외 | 2,800자 이상 (목표 3,500~5,000, 절대 하한 2,500 — 채우기용 문장으로 도달 금지) |
| 제목 공백 포함 | 22~35자 (직접 세기, 추정 금지 — 모바일 35자 초과 시 잘림) |

### 키워드

| 항목 | PASS |
|---|---|
| 메인 키워드 본문 등장 | 5~8회 |
| 메인 키워드 제목 등장·맨 앞 | True |
| 한 문장 내 메인 키워드 2회+ | 0건 |
| 서브 키워드 각 등장 | 2~4회 |

### 구조·CTA

| 항목 | PASS |
|---|---|
| 소제목 개수 | 4~6개 |
| 평균 단락 문장 수 | 2~4문장 |
| 본문 중간 CTA (50~70%) | 1회 이상 |
| 마지막 CTA | 1회 |
| "무료" 단독 사용 | 0회 |

## Step 2. 절대 금지 스캔

아래 패턴이 1건이라도 발견되면 즉시 재작성 대상(FAIL).

**메타 표기**: `[제목]:` `[소제목]` `---` `##` ` ``` ` `**` 그대로 노출

**블로거 톤**: 안녕하세요 / 여러분 / 독자분들 / 저는·제가 / 오늘은~알아보겠습니다 / 함께 살펴보시죠 / 도움이 되셨길 / 이웃추가 / 공감과 댓글

**격언체**: "숫자는 정직하다" / "결과는 같았다" / "그게 진짜 비용이다" / "원인은 단절이다"

**광고 표현**: 무료 진단·체험 / 100% 만족 / 강력 추천 / 지금 바로 클릭

**경쟁사 자료 인용**: CS쉐어링과 동일 카테고리 서비스를 제공하는 기업명·서비스명·블로그명이 출처로
본문에 등장(예: tryvox/vox.ai) — 기관명을 지운 "업계 조사에 따르면" 식 순화 표현도 원 수치가
경쟁사 자료에서 왔다면 동일하게 FAIL

아래 4개는 앞의 항목들과 달리 **정확한 문자열 매칭이 아니라 판단 기준**이다. 예시 문구와 토씨가
달라도 같은 문제면 FAIL, 예시와 표면이 비슷해도 문제 유형에 해당하지 않으면 PASS로 본다.

**구어체 결론(데이터 서술 문장)**: 통계·조사 결과를 설명하는 문장이 구어체 어미로 끝나 격식이
무너지는가 (예: ~인 셈이다 / ~인 거죠. 도입 훅 공감 문장·CTA 권유형은 예외 — `01-writing-guide.md` §4)

**예고성 전환구**: 다음 내용을 예고만 하고 정보·판단이 없어 빼도 무방한 문장인가 (예: ~도 눈여겨볼
대목입니다 / 표로 보면 ~드러나네요 — `01-writing-guide.md` §8)

**타사 인물 실명**: 회사명이 아닌 타사 대표·임원 개인 이름이 주어로 등장하는가

**AI 완전 대체 뉘앙스**: AI+상담사 협업 프레임 없이 자동화가 상담사를 전부 대체한다고 읽히는가 (예:
사람 없이 해결된다 / 상담사가 필요 없어진다 — `05-brand-cta-reference.md` 4절 참고)

**인용 밀도 과다**: 문장마다 출처가 붙어 논문처럼 읽히는가 — 소제목 하나당 출처 인용이 1~2개를
넘고, 그중 주장을 세우는 데 실제로 안 쓰이는(빼도 흐름에 지장 없는) 인용이 있는가 (`01-writing-
guide.md` §8 참고)

## Step 3. 리캐치 스타일 검증

| 항목 | PASS |
|---|---|
| 도입 첫 줄 이모지 (🤔/💭/🎯/💡 등) | 1개 |
| 도입 훅 패턴 (질문/공감/장면/수치) | 1개 이상 |
| 도입 분량 | 250~350자 |
| 목차 "📑 이 글의 순서" 표기 | True |
| 목차 항목 개수 | 3~5개 |
| 목차와 본문 소제목 일치 | True |
| 글 전체 이모지 개수 | 5~8개 |
| 한 단락 이모지 | 1개 이하 |
| 감정·B2B 부적합 이모지 (😂🥳🎉🔥 등) | 0개 |
| 같은 이모지 3회+ 반복 | 0개 |
| 격식체 비율 | 65~85% |
| 친근체 비율 ("~해요/~죠/~거든요/~네요") | 15~35% |
| 한 톤 90% 이상 | FAIL |
| 명사구로 시작하는 단락 비율 | 80% 이하 |
| "첫째~둘째~셋째" 정형 병렬 남발(불필요한데 사용) | 정성 -1점 |
| 한 단락 접속사 2회 이상 | 정성 -1점 |
| "~했고요/~고요"류 나열형 어미 사용 | 정성 -1점 |

**"첫째~둘째~셋째" 판단**: 병렬 원인·이유가 3개 이상이라 표지 없이는 항목이 헷갈리는 경우에 쓴
것이면 감점하지 않는다. 2개 이하이거나 자연스러운 접속만으로 충분한데도 습관적으로 쓴 경우만 감점
대상이다 (`01-writing-guide.md` §6 참고).

**"~했고요/~고요" 판단**: §4 표에서 허용하는 친근체(~죠·~해요·~거든요·~네요)에 속하지 않는 별도
문제다 — 여러 사실을 캐주얼하게 죽 이어붙이는 나열형 말투로 읽히면 감점, "~했습니다/~입니다"로
바꿔도 자연스러운데 습관적으로 썼으면 감점 대상이다 (`01-writing-guide.md` §4 참고).

## Step 4. 정성 항목 셀프리포트

모호하면 낮은 점수 (4 vs 3 → 3, 5 vs 4 → 4). 자기 출력에 후하게 매기지 말 것.

| 항목 | 5점 기준 | 3점 기준 | 1점 기준 |
|---|---|---|---|
| 검색 의도 부합도 | 0.5초 내 "내 문제다" 인식 | 가치 제안 모호 | 검색 상황 파악 어려움 |
| 일반론 회피도 | 모든 단락에 수치·사례 | 절반 단락에 일반론 | 누구나 쓸 수 있는 글 |
| 종결 어미 혼용 | 격식 65~69% (이상적) | 격식 86~94% | 단일 톤 100% |
| 도입 훅 자연스러움 | 이모지+공감+자연스러운 진입 | 이모지 또는 공감 한쪽 누락 | "안녕하세요" 또는 즉시 본론 |
| 구체 수치·고유명사 밀도 | 모든 단락에 수치·업종·기간 1개+ | 60% 단락에 등장 | 거의 없음 |
| 상업적 설득력 | 모든 소제목이 문제→서비스 해결 | 2개 소제목 일반론으로 끝남 | 시장 리포트처럼 읽힘 |
| 발행 가능성 | Step 0 PASS + 그대로 발행 가능 | 3~5곳 수정 필요 | Step 0 FAIL / 재작성 권장 |

**평균 4점 미만 → 재검토 필수. Step 0 FAIL → 발행 가능성 자동 1점.**

종결 어미 4점 권장 범위: 격식 70~85%.

---

## Step 5. 종합 점수

```
정량 점수 = (통과 항목 / 전체) × 100
정성 점수 = (평균 / 5) × 100
종합 점수 = (정량 × 0.6) + (정성 × 0.4)
```

| 점수 | 등급 | 처리 |
|---|---|---|
| 95~100 | A+ | PASS |
| 88~94 | A | PASS |
| 80~87 | B | PASS + 보완 권장 사항 메모 |
| 70~79 | C | FAIL — 재작성 권장 |
| 70 미만 | F | FAIL — 재작성 |
