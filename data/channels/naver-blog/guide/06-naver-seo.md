# 06. SEO · AEO · GEO 가이드 (네이버 + AI 검색)

> **역할**: 네이버 C-Rank/D.I.A+ 및 AI 검색(AEO/GEO) 최적화의 단일 기준.
> **네이버는 정확한 알고리즘 임계값(글자수·체류시간 등 구체 숫자)을 공식적으로 공개하지 않는다** —
> 아래 수치는 실전에서 관찰된 경향을 정리한 휴리스틱이며, 절대 기준이 아니다.
> 분량·제목·이미지 등 글쓰기 수치의 원 출처는 `01-writing-guide.md` 2절이다 — 이 문서와 숫자가
> 어긋나면 01을 따르고 이 문서를 갱신한다.

---

## 1. 핵심 알고리즘 2개 — C-Rank + D.I.A+

네이버 블로그 검색 순위는 두 알고리즘이 **함께** 결정한다. 하나만 높아도 부족하다.

### C-Rank (채널 신뢰도)
> "이 블로그가 이 분야의 전문 채널인가?"

| 평가 축 | 설명 | 실전 적용 |
|---|---|---|
| **Context (주제 일관성)** | 블로그 전체가 특정 분야에 집중되어 있는가 | 한 주제(CS·CX·B2B)에만 발행. 잡블로그 = 패널티 |
| **Content (콘텐츠 누적)** | 해당 주제의 포스팅이 충분히 쌓였는가 | 동일 카테고리 최소 30~50개 이상이 전문성 인정 기준 |
| **Chain (사용자 반응)** | 방문자가 얼마나 많이, 깊게 소비하는가 | 체류시간·재방문·공유가 Chain 점수에 직결 |

### D.I.A+ (Deep Intent Analysis Plus — 문서 품질)
> "이 글이 검색 의도를 정확히 충족하고, 독창적 경험을 담고 있으며, 독자를 붙잡는가?"

| 평가 축 | 설명 | 실전 적용 |
|---|---|---|
| **검색 의도 적중** | 제목·소제목·본문이 실제 검색어와 맥락이 일치하는가 | 제목에 핵심 키워드 맨 앞 배치. 소제목 중 최소 1개는 질문형 |
| **독창적 경험** | 직접 경험·자체 수치·실명 사례가 있는가 | 일반론 나열 문서는 D.I.A+가 낮게 평가 |
| **콘텐츠 구조** | 목차·소제목·리스트가 명확해 AI가 파싱 가능한가 | 정보 블록화: 표·목차·요약 박스 적극 활용 |
| **사용자 행동 신호** | 체류시간·스크롤 깊이·재방문·공유가 높은가 | 목표 체류시간 2분 30초~3분 이상 |

## 2. AI·AEO·GEO

2026년부터 네이버 검색은 단순 문서 나열을 넘어 **AI가 먼저 읽고 요약·추천**하는 구조로 전환됐다.

### AEO (Answer Engine Optimization)
- "AI가 답변을 생성할 때 이 글에서 인용하게 만들기"
- 각 섹션 첫 단락을 **80자 이내 핵심 요약문**으로 시작 → AI가 발췌하기 쉬운 구조.
- 질문형 소제목 + 직접 답변 단락 조합이 AEO에 최적.
- **앞쪽 배치가 특히 중요하다**: AI 검색 인용의 상당수가 문서 앞쪽 30% 구간에서 나온다. 결론·핵심
  수치를 도입부 직후에 먼저 압축해서 배치한다.

### GEO (Generative Engine Optimization)
- ChatGPT·Perplexity 등 외부 AI 검색에서도 인용되는 문서 만들기.
- 실명 데이터·수치·사례가 있어야 AI가 신뢰 소스로 선택.
- **엔티티 중심 구성**: "CS쉐어링" · "OASIS AICC+IPCC" 같은 고유 개체를 문서 안에서 일관된 이름으로
  반복 등장시킨다. 표기가 흔들리면(예: 같은 서비스를 다른 이름으로 부름) 인식이 약해진다.
  `05-brand-cta-reference.md` 카탈로그 표기 고정 규칙이 이 목적도 겸한다.

### 통합탭 누락 리스크
- 품질 낮은 블로그 글은 통합탭 자체에서 누락될 수 있다.
- 누락 예방: 충분한 본문 분량(2,800자↑, 채우기용 아닌 실제 정보) + 고유 경험 + 이미지 6~8장 + 체류시간 확보.

## 3. 실전 최적화 체크리스트

### 제목 최적화
| 항목 | 기준 |
|---|---|
| 글자수 | 공백 포함 **22~35자** (모바일 35자 이상 잘림) |
| 키워드 위치 | 핵심 키워드를 **맨 앞**에 배치 |
| 검색 의도 | "~방법", "~이유", "~비교", "~사례" 등 실제 검색 패턴과 일치 |
| 금지 표현 | "무조건", "100%", "최고", "업계 1위" 등 광고성 단어 → 신뢰도 감점 |

### 본문 최적화
| 항목 | 기준 |
|---|---|
| 분량 | **2,800자 이상** (목표 3,500~5,000자). 2,500자 미만 = 통합탭 누락 리스크 |
| 키워드 배치 | 제목·첫 단락·소제목·이미지 캡션에 자연스럽게 분산. 도배 금지 |
| 소제목 구조 | 4~6개, 그중 1개 이상 질문형 |
| 단락 길이 | 단락당 100~180자, 3~4문장 |
| 목차 | 필수. 소제목과 1:1 연결 |

### 이미지 최적화
| 항목 | 기준 |
|---|---|
| 수량 | **6~8장**. 개수 자체보다 "핵심을 짚는 시각자료인가"가 더 중요하다 |
| 캡션 | 필수. 설명형 + 핵심 키워드 1회 자연스럽게 포함 |

### 체류시간 확보 전략
목표: **2분 30초~3분 이상**
1. 도입부에서 "이 글을 끝까지 읽어야 하는 이유"를 약속
2. 목차로 스크롤 동선 안내
3. 짧은 단락 + 이미지 → 스크롤 호흡 유지

## 4. 키워드 선정 기준 (research 단계 참고)

### 좋은 키워드 조건
| 조건 | 설명 |
|---|---|
| **검색 의도 명확** | "CS 비용 절감" → 해결책 찾는 실무자. 의도가 명확할수록 전환율 높음 |
| **경쟁 강도 적정** | 월 검색량 1,000~10,000, 상위 노출 난이도 낮음 |
| **롱테일 우선** | "콜센터 아웃소싱 비용" > "콜센터" — 구체적 키워드가 전환 품질 높음 |
| **주제 연관성** | CS·CX·B2B 고객서비스 범주 내 키워드만 선택 |

### 키워드 배치 규칙
- 제목: 핵심 키워드 1개 맨 앞
- 첫 단락: 핵심 키워드 + 연관 키워드 1개 자연스럽게 삽입
- 소제목: 롱테일 키워드 2~3개 분산
- 이미지 캡션: 키워드 1회씩
- 과다 반복(3회 이상 동일 문장 내) = D.I.A+ 감점
