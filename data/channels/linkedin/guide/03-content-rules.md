# 내용/규칙 검수 rubric

검수(content-review)는 **A. 자동검수(binary)** → **B. 7지표 채점(1~5점, 일부 0~5점)** → **C. 임계 판정** 순으로 판정하고, reviewer.md의 JSON 형식으로 출력한다. writer는 이 rubric을 **선제 준수**한다(제출 전 자가 채점).

---

## A. 자동검수 (binary — 하나라도 위반 시 `verdict: "FAIL"`, 점수와 무관)
위반 항목명을 `auto_check_violations` 배열에 넣는다.

1. **특수기호 노출**: 본문에 `\n`·`\t`·`\r`·`\\n` 같은 이스케이프 시퀀스가 텍스트로 노출(플레인 텍스트는 실제 줄바꿈이어야 함).
2. **출처 날조**: 통계·수치·퍼센트·연구/조사 결과를 인용하면서 출처 미표기 또는 날조(예: "…40% (출처: SQM Group, 2025)" 형태로 출처·연도 명시돼야 함). → 사실_정확성 0점과 연동.
3. **경쟁사 언급**: 메타콜, KT CS, KT IS, KTis, 유베이스, 트랜스코스모스코리아, CJ텔레닉스, 롯데컨택센터, 효성ITX, 현대HCN, 인성데이터콜 등 경쟁사 브랜드/서비스명 직접 언급.
4. **브랜드 오표기**: 정확 표기는 **CS쉐어링**(또는 "CS Sharing"). 오표기 반려: 씨에스쉐어링, cs쉐어링(소문자), CS쉔어링, CS쉐링, 씨에스 쉐어링.
5. **필수 요소 누락**: remind(핵심 압축 한 문장) / 소통 유도(CTA) 문구 / 해시태그(마지막에 `#CS쉐어링`) 중 하나라도 없음.
6. **해시태그 오표기**: `해시태그#태그명`처럼 "#" 앞에 글자를 붙이는 잘못된 표기.
7. **주제 이탈**: CS/CX/AICC/AX 도메인에서 벗어남. 핵심 도메인 키워드(CS, CX, 고객, 상담, 서비스)가 3개 미만.
8. **타깃 오류**: "개인 셀러/1인 쇼핑몰/사장님 혼자" 등 소비자·개인 타깃으로 읽힘(02-personas 참조).

---

## B. 7지표 채점 (각 1~5점, 사실_정확성·논리성만 0~5점)

| 지표 | 점수 | 5점 기준 | 3점 이하 사유 |
|---|---|---|---|
| **문장_자연스러움** | 1~5 | 문법·가독성 매끄럽고 리듬 좋음 | 어색한 번역투·비문·중언부언 |
| **사실_정확성** | 0~5 | 수치·서비스 설명이 정확, 출처 명시 | 출처 없는 수치·CS쉐어링 서비스 날조 = **0점(즉시 FAIL)** |
| **타깃_문제_반영** | 1~5 | 02-personas 1명 겨냥, **그 페르소나의 pain 트리거 키워드가 본문에 실제로 등장** | 주제만 관련 있고 페르소나 특유의 언어가 안 보이면 3점 (02-personas 채점 기준 적용) |
| **채널_적합성** | 1~5 | 04-structure 블록 순서 + 00-tone 수치 목표(글자수 1000~1200자) 충족. 시각화 표현을 맥락에 맞게 활용했으면 가점 | 구조 위반·글자수 이탈(700~1600 밖). **시각화 표현 사용 자체는 감점 사유가 아니다** — 억지로 개수를 맞추려 내용 없이 채운 경우만 감점 |
| **후킹력** | 1~5 | 통념→반전이 선명, 인사이트가 깊음 | **"~한데 ~인지"류 안전한 반문형이면 3점 이하.** 반전 없는 일반 결론, "OO의 무덤/배신/역설"류 클리셰 은유도 ≤3 |
| **CTA_연결성** | 1~5 | remind→소통유도가 본문과 논리적으로 연결 | 기계적·본문 무관한 CTA, 매번 같은 상투 문구 |
| **논리성** | 0~5 | 통념→반전→원인→해법→remind가 비약 없이 이어지고, **해법이 원인의 핵심 키워드를 다시 언급하며 명시적으로 연결됨** | 원인 나열 후 해법이 무관한 일반론으로 붕 뜸, 근거 없는 결론 비약 = **0점(즉시 FAIL)** |

---

## C. 임계 판정 (합계 0~35)
- 자동검수 위반이 하나라도 있음 **또는** 사실_정확성 = 0 **또는** 논리성 = 0 **또는** 합계 < 24 → `verdict: "FAIL"` (엔진이 feedback으로 재작성)
- 합계 **24~27** → `verdict: "조건부 PASS"` (통과 처리 — 엔진은 재작성하지 않음)
- 합계 **≥ 28** → `verdict: "PASS"`

> 엔진(isRejected)은 `verdict` 값에 reject/fail이 포함될 때만 재작성한다. `"조건부 PASS"`·`"PASS"`는 그대로 통과한다. 그러므로 반려는 반드시 `verdict: "FAIL"`로 낸다.

출력은 reviewer.md의 JSON 스키마(scores·total·auto_check_violations·verdict·feedback)를 정확히 따른다.
