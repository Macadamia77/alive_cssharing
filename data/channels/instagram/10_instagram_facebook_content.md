# Instagram / Facebook 콘텐츠 규칙

## 카드뉴스
- 카드 수는 방향의 story_plan 수에 따라 총 5장 또는 6장으로 작성한다.
- 중간 카드에 필요한 서로 다른 메시지가 3개면 후킹 1장, 중간 카드 3장, CTA 1장으로 총 5장으로 구성한다.
- 중간 카드에 필요한 서로 다른 메시지가 4개면 후킹 1장, 중간 카드 4장, CTA 1장으로 총 6장으로 구성한다.
- 카드 수를 늘리기 위해 같은 내용을 표현만 바꿔 반복하지 않는다.
- 첫 장 후킹은 06_hook_pattern_library.md의 7가지 유형(통계·수치 제시형, 반문형, 자기고백/공감형, 경고·손실회피형, 비교·재정의형, 대상 지정형, FOMO·트렌드형) 중 이번 주제에 가장 잘 맞는 유형을 매번 새로 판단해서 고르는 것을 기본으로 한다.
- "~한데 ~인 이유", "~쌓이는데 ~막막한 이유"처럼 상황과 결론을 단순 나열만 하고 끝내는 안전한 문제 제기형 문장을 기본값으로 쓰지 않는다. 이 패턴은 7가지 유형이 전부 주제에 안 맞을 때만 최후 수단으로 쓴다.
- 자기고백/공감형을 쓸 때는 담당자가 스스로를 떠올리게 하는 상황·감정 묘사이되, 대상을 소비자·개인 셀러가 아닌 B2B CS 담당자로 유지한다.
  - 예: "쌓이기만 하는 VOC, 확인은 하고 계세요?", "그 문의, 이번 달에도 반복됐다면"
- 06_hook_pattern_library.md의 예시 문구를 그대로 베끼지 않고 실제 주제에 맞게 새로 쓴다.
- 마지막 장은 새로운 문제를 추가하지 않고, 서비스 확인 또는 운영 점검 CTA 하나로 마무리하는 것을 기본으로 한다.
- 참여 유도가 필요한 주제에서는 댓글 유도형 CTA도 사용할 수 있다.
  - 예: "댓글 남겨주시면 관련 자료를 DM으로 보내드려요"
  - 이때 제공하는 자료는 미리 준비된 고정 자료 하나여야 한다. "업종에 맞는", "귀사에 맞춤" 등 댓글마다 개별 제작이 필요한 표현은 쓰지 않는다.
- first와 CTA 카드의 강조색·스타일은 Figma 템플릿을 유지하므로 highlight_text를 비운다.
- 중간 카드 title은 반드시 2줄로 작성한다.
- title은 줄당 최대 10자 내외로 작성한다.
- 중간 카드 subtitle은 반드시 1줄이며 최대 26자 내외로 작성한다.
- title이 2줄일 때 줄바꿈은 글자 수를 맞추기 위해 임의 위치에서 끊지 않고, 주어/서술어 또는 수식어/명사 같은 의미 단위 경계에서 끊는다.
- 일반 박스 item.title은 최대 10자 내외로 작성한다.
- 일반 박스 item.body는 최대 25자 내외로 작성한다.
- 비교형 박스 item.title은 최대 8자, item.body는 최대 20자 내외로 작성한다.
- 문구가 길어질 경우 문장을 자르지 말고 처음부터 더 짧게 다시 작성한다.
- steps_vertical은 실제 순서, 단계, 진행 흐름이 있을 때만 사용한다.
- 하나의 카드뉴스 안에서 같은 layout_type은 최대 2회까지만 사용한다.
- 병렬 사례·조건·확인 항목은 steps_vertical 대신 stacked_boxes 또는 keyword_boxes를 우선 사용한다.

### 레이아웃 선택 규칙

* 레이아웃은 장식용으로 무작위 선택하지 않고, 카드 안 정보의 관계에 따라 선택한다.
* steps_vertical은 실제 순서, 점검 단계, 실행 단계가 있을 때만 사용한다.
* flow_process는 앞 단계가 다음 단계로 이어지는 실제 처리 흐름이 있을 때만 사용한다.
* compare_2col은 두 대상, 두 기준, 이전과 이후처럼 정확히 2개를 비교할 때 사용한다.
* keyword_boxes는 같은 축의 항목 3~4개를 병렬로 정리할 때 사용한다.
* stacked_boxes는 반복 신호, 예외 상황, 확인 항목처럼 독립된 정보 2~3개를 정리할 때 사용한다.
* 중간 카드가 3장인 경우 최소 2개 이상의 layout_type을 사용한다.
* 중간 카드가 4장인 경우 최소 3개 이상의 layout_type을 사용한다.
* 같은 layout_type을 연속된 중간 카드에 사용하지 않는다.
* 내용의 관계에 맞지 않는 레이아웃을 억지로 사용하지 않는다.

## Instagram / Facebook 캡션 규칙

- 카드 문구를 순서대로 복사하지 않고, 게시물 전체 관점과 점검 포인트를 요약한다.
- 모바일에서 읽기 쉽게 문단 사이 빈 줄을 사용한다.
- caption에는 이모지를 총 3~5개 사용한다.
- 첫 문장은 주제를 환기하는 한 줄로 시작하며, 문장 맨 앞에 💡, 📌, 🔎, ❓ 중 내용에 맞는 이모지 1개를 사용한다.
- 서비스 방식, 운영 기준, 비교 포인트처럼 한 번 더 강조할 내용이 있을 때는 📌를 1~2회 추가로 사용할 수 있다.
- CTA 문단에는 이모지를 반드시 넣을 필요는 없으며, 필요할 때만 👉를 사용한다.
- 😅, 😊, 🤔 등 감정 이모지도 문맥상 자연스러우면 사용할 수 있으나, 핵심 내용의 시작 표시는 ✅, ✔️, 📌을 우선 사용한다.
- 같은 이모지를 모든 문단에 반복하거나, 모든 문장 끝에 이모지를 붙이지 않는다.
- 해시태그는 caption에 넣지 않고 hashtags 배열에 분리한다.
- hashtags는 12~15개이며 #CS쉐어링 #CS대행 #고객센터대행을 포함한다.
- caption은 해시태그를 제외하고 약 450~650자 내외로 작성한다.
- caption은 도입 / 문제 맥락 / 핵심 포인트 2~3개 / CTA 순서로 작성한다.
- 각 문단은 빈 줄로 구분한다.
- 핵심 포인트 문단은 한 줄 요약으로 끝내지 말고, 왜 그런 문제가 생기는지 또는 담당자가 무엇을 확인해야 하는지 1~2문장으로 설명한다.

## 중간 카드 흐름 구조

- 중간 카드 흐름은 주제 성격에 따라 아래 패턴 중 가장 자연스러운 것을 선택한다.
- 3장: 문제 현상 → 요인 분석 → 해결 방안 (기본)
- 4장 A: 문제 현상 → 요인 분석 → 해결 방안 → CS쉐어링 적용 방식 (서비스 연결이 자연스러울 때)
- 4장 B: 문제 현상 → 요인 분석 1 → 요인 분석 2 → 해결 방안 (요인이 복잡할 때)
- 4장 C: 문제 현상 → 요인 분석 → 해결 방안 1 → 해결 방안 2 (해결책이 2단계일 때)
- 각 카드 내 항목은 해당 카드의 축(현상/요인/해결책) 하나로 통일하며, 현상과 해결책을 한 카드에 섞지 않는다.
- 요인 분석 카드에서 해결 방안 카드로 넘어갈 때, 해결 방안 카드의 title은 앞 카드에서 다룬 요인을 해결하는 방향으로 자연스럽게 이어진다.

## 문장 톤
- 도입은 가볍게 공감시키되, 문제 설명과 해석은 B2B 실무형으로 정리한다.
- 카드 title에는 이모지를 사용하지 않는다.
- item.title은 4~14자, item.body는 한두 줄로 읽히는 설명형 문장으로 작성한다.
- title, subtitle, body는 개념을 명사로 나열해 붙이지 않고, 주어와 서술어가 자연스럽게 이어지는 문장으로 작성한다.
  - 금지 예: 성수기 피로감에 더해 비수기 문의의 특성 (명사구+명사구 나열, 의미가 끊김)
  - 권장 예: 성수기 피로가 채 가시기도 전에 비수기 특유의 문의까지 겹치면서 (주어-서술어로 자연스럽게 연결)
- 문장을 소리 내어 읽었을 때 어색하면, 글자 수를 맞추기 위해 조사나 서술어를 생략하지 말고 표현 자체를 다시 쓴다.
