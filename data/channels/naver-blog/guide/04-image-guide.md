# 04. 이미지 카드 가이드

> **역할**: 이미지 카드 타입별 콘텐츠 스키마(JSON 필드·글자수 규칙).
> 이미지를 배치할지 말지의 판단 기준(개수·위치)은 writer 단계 책임(`01-writing-guide.md` 9절) —
> 이 문서는 "카드에 어떤 내용을 채우는가"만 다룬다.
> 글 한 편당 카드 총 수량: **4~6장** (썸네일 1 + 본문 카드 N).
> **2026-07-10 개편**: 카드는 더 이상 HTML/CSS 마크업으로 직접 작성하지 않는다. image-gen은
> 정해진 JSON 필드에 짧은 텍스트만 채우고, 실제 도형·색상·좌표는 코드(`cardTemplateBuilder.ts`)가
> 결정적으로 SVG로 조립한다 — 그 뒤 서버가 PNG로 래스터화해 본문에 삽입하고, 원본 SVG는 따로
> 저장해 다운로드(Figma/일러스트레이터 편집용)로 제공한다. 색상·규격 수치의 단일 소스는 이제
> `cardTemplateBuilder.ts` 코드다 — 이 문서는 값이 아니라 "채워야 할 필드가 무엇인지"만 설명한다.

---

## 1. 대표 카드 (썸네일) — image-gen이 작성하지 않는다

**첫 번째 `[IMAGE: ...]` 마커(대표 썸네일)는 image-gen이 손으로 만들지 않는다.** 색상·레이아웃이
매번 조금씩 달라지며 반복적으로 문제가 생겨서(디자인 편차, 마커 문법 혼동으로 인한 텍스트 노출 등),
`cardTemplateBuilder.ts`의 `buildThumbnailSvg()`가 draft의 제목·부제와 `assets/mascot.png`만으로
결정적으로 조립한다. **image-gen에게 오는 요청에는 이미 이 사실이 명시되어 있으므로, 첫 번째
마커분 카드는 작성하지 말고 두 번째 마커부터 작성한다.**

디자인 자체를 바꾸고 싶으면(색상·레이아웃 등) `src/lib/pipeline/cardTemplateBuilder.ts`의
`buildThumbnailSvg()` 함수를 직접 수정한다 — 코드 변경이라 재배포가 필요하다(이전엔 별도 HTML
템플릿 파일 하나만 고치면 됐지만, 진짜 벡터 SVG로 좌표를 계산해 조립하는 구조라 코드 안에
값이 있다).

## 1-1. B2B 맥락 필수 준수 (절대 규칙, 모든 카드 공통)

이 블로그는 B2B 마케팅 콘텐츠다. 독자는 기업 담당자(운영팀장·CS팀장·대표)이며, 목표는 "CS쉐어링
서비스를 도입해야겠다"는 판단을 유도하는 것이다.

- **허용**: 콜센터 상담 화면·헤드셋·CRM 대시보드, 업무 그래프·KPI 수치 카드, 비용 비교표,
  플로우차트(CS 운영 흐름), 기업 담당자 아이콘, CS쉐어링 서비스 명칭·로고
- **금지**: 소비재 제품 이미지(에어컨·선풍기·가전·의류 등), 소비자(B2C) 쇼핑 장면, 계절
  풍경·날씨 아이콘, 일반 생활 사진 소재
- 카드 안에 들어가는 텍스트는 모두 **CS 업무·콜센터·기업 운영** 맥락으로만 구성한다.

## 2. 본문 카드 — JSON 콘텐츠 스키마

**모든 소제목마다 카드를 만드는 것이 목적이 아니다.** 카드는 시각적 요약·강조가 실제로 도움이 되는
지점에만 선택적으로 배치한다 (전체 카드 4~6장 예산 안에서 배분).

- **비교표·수치 나열형 정보**(요금·항목별 대조·전후 수치 등)는 카드로 만들지 않고 **본문 마크다운
  표**로 작성하도록 writer 단계에서 이미 처리되어 있어야 한다 — `table` 레이아웃은 항목이 2~3개뿐
  이라 본문 표보다 카드 한 장이 더 임팩트 있는 경우에만 예외적으로 쓴다.
- **카드 하나만 봐도 내용이 이해되게 쓴다.** 본문 문장을 그대로 잘라 붙이지 말고, 카드 자체가
  독립적으로 완결된 메시지(주장 + 핵심 수치/이유)를 전달하도록 카피를 새로 쓴다.
- **글자수 제한을 넘기면 코드가 자동으로 자르거나(말줄임) 줄바꿈한다** — 넘치지 않게 처음부터
  짧게 쓰는 게 가장 안전하다. 아래 표의 글자수는 공백 포함 기준이다.

### 2-0. 모든 카드 공통 필드

| 필드 | 설명 | 글자수 |
|---|---|---|
| `layout` | 아래 5종 중 하나: `"summary"`/`"numbered"`/`"chat"`/`"badges"`/`"table"` | - |
| `contextLabel` | 상단 컨텍스트 바 — 글 제목 축약 | ≤20자 |
| `headline` | `[1행, 2행]` — 1행은 맥락 설명(검정), 2행은 핵심 메시지(네이비블루) | 각 행 ≤16자 |
| `subtext` | 보조 설명 1줄 (선택 필드, 생략 가능) | ≤50자 |
| `cta` | `[1행, 2행]` — 행동 유도 문장 | 각 행 ≤26자 |

### 2-1. 레이아웃별 전용 필드 (5종 — 내용에 맞게 선택)

| 타입 | 사용 시점 |
|---|---|
| `summary` | 시각 요약이 도움이 되는 소제목에 선택적으로 |
| `numbered` | Why?·원인·단계 3~5개 나열 |
| `chat` | 고객 발화·내부 대화 시나리오 묘사 |
| `badges` | 특장점·운영 옵션 조합 표현 |
| `table` | 항목이 2~3개뿐이라 본문 마크다운 표보다 카드 한 장이 더 임팩트 있는 경우만 |

**`summary`**
```json
{ "layout": "summary", "contextLabel": "...", "headline": ["...", "..."], "cta": ["...", "..."],
  "body": "소제목의 핵심 메시지 — 구체적 수치 포함, 최대 3줄까지 자동 줄바꿈(총 120자 내외)" }
```

**`numbered`** — `items` 3~5개
```json
{ "layout": "numbered", "contextLabel": "...", "headline": ["...", "..."], "cta": ["...", "..."],
  "items": [
    { "title": "항목 제목(≤14자)", "desc": "이유 설명(최대 2줄 자동 줄바꿈, 줄당 28자 내외)" }
  ] }
```

**`chat`** — `bubbles`는 정확히 2개(고객 발화 1·2), 그다음 `conclusion`(핵심 결론)으로 이어진다
```json
{ "layout": "chat", "contextLabel": "...", "headline": ["...", "..."], "cta": ["...", "..."],
  "bubbles": ["고객 발화 1(최대 2줄, 줄당 26자 내외)", "고객 발화 2"],
  "conclusion": "핵심 결론 1~2줄(최대 2줄, 줄당 30자 내외)" }
```

**`badges`** — `tags` 2~4개, `gridItems`는 정확히 2개
```json
{ "layout": "badges", "contextLabel": "...", "headline": ["...", "..."], "cta": ["...", "..."],
  "tags": ["태그(≤10자)", "태그2"],
  "gridItems": [
    { "title": "특징 A(≤12자)", "desc": "설명(최대 2줄, 줄당 16자 내외)" },
    { "title": "특징 B", "desc": "설명" }
  ] }
```

**`table`** — `columns` 2개(비교 대상), `rows` 2~3개
```json
{ "layout": "table", "contextLabel": "...", "headline": ["...", "..."], "cta": ["...", "..."],
  "columns": ["대상 A(≤10자)", "대상 B"],
  "rows": [
    { "label": "항목 1(≤14자)", "values": ["값(≤10자)", "값"] }
  ] }
```

## 3. 공통 제작 규칙

| 항목 | 규칙 |
|---|---|
| 제작 방식 | image-gen은 JSON만 작성 — 실제 SVG 조립은 `cardTemplateBuilder.ts`가 결정적으로 수행 |
| 폰트·배경·강조색·카드 크기·여백 스케일 | 전부 `cardTemplateBuilder.ts` 코드에 고정값으로 있음(가이드에 값 중복 기재 안 함) |
| 카드 높이 | 항목 수(리스트 개수 등)에 따라 코드가 자동 계산 — LLM이 높이를 신경 쓸 필요 없음 |
| 다운로드 포맷 | PNG(본문 삽입·붙여넣기용) + SVG(Figma/일러스트레이터 편집용) 둘 다 제공 |
| 썸네일 크기 | 720 × 720px (`buildThumbnailSvg()`) |
